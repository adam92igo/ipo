# IPO Compass

SaaS platform that guides European SMEs through IPO readiness: weighted diagnostic questionnaire, deterministic valuation (DCF, comparables, market multiples), rules-based roadmap, and AI-assisted company profiling. MVP scope: France-first content (Pappers registry, Euronext Paris), English UI.

## Stack

- Next.js 15 (App Router) + TypeScript + Tailwind v4 + shadcn/ui (`src/components/ui`)
- PostgreSQL 17 (local via `docker compose up -d`) + Drizzle ORM (node-postgres)
- better-auth with the `organization` plugin — multi-tenant: one organization owns N companies, roles `owner` / `admin` / `member`
- Anthropic API (claude-sonnet) for AI modules ONLY (module 5) — never inside scoring, valuation, or roadmap engines
- Vitest for tests

## Commands

- `pnpm dev` — dev server (Turbopack)
- `pnpm test` / `pnpm test:watch` — Vitest (integration tests need Docker Postgres running)
- `pnpm typecheck` — `tsc --noEmit`
- `pnpm lint` — ESLint
- `pnpm db:generate` / `pnpm db:migrate` — Drizzle Kit migrations (never edit generated SQL by hand)
- `pnpm auth:schema` — regenerate `src/db/schema/auth.ts` from the better-auth config (run after adding auth plugins)

## Architecture

- `src/db/schema/auth.ts` — GENERATED by better-auth CLI; do not hand-edit
- `src/db/schema/app.ts` — business tables: company, company_financials, assessment, answer, valuation_run, roadmap_item
- `src/lib/auth.ts` / `src/lib/auth-client.ts` — better-auth server/client instances
- `src/lib/data-access/` — the ONLY place business tables may be queried
- `src/engines/` — pure deterministic engines (scoring, valuation, roadmap rules): no I/O, no DB, no framework imports; fully unit-tested
- `config/` — versioned business content: `questionnaire.v*.json`, `roadmap-rules.v*.json`, `valuation-refs.v*.json`. Content lives here, never hardcoded and never in DB. Stored records always reference the version they were computed with.

## Non-negotiable invariants

1. **Tenant isolation**: every business table carries `organization_id`. All reads/writes go through `src/lib/data-access/`, which requires an org context resolved from the session (`requireOrgContext`). Never write a raw Drizzle query in a route, server action, or component. Never accept an `organizationId` from client input — it always comes from the session.
2. **Determinism**: scoring, valuation, and roadmap generation are pure functions in `src/engines/` developed TDD (tests written first). No AI, no network, no randomness, no dates from `Date.now()` inside engines — pass inputs explicitly.
3. **Validation**: every mutation validates input with Zod at the boundary (server action / route handler) before touching data-access.
4. **Secrets** stay in `.env` (gitignored); `.env.example` documents required variables without values.
5. UI copy is English. Amounts are EUR.

## Module roadmap

1. Foundations (auth, schema, dashboard layout) — this module
2. Diagnostic engine (questionnaire, weighted scoring, radar restitution) — TDD
3. Valuation (DCF, sector comparables, market multiples; aggregated min–max range) — TDD
4. Roadmap (rules engine from assessment results)
5. AI modules ("Fill with AI" from official website + Pappers only — no LinkedIn scraping; contextual IPO assistant)

One module at a time; clean commit + code review before moving to the next.
